Introduction to Mobile App Development

Welcome to the world of app development! Every day, millions of people use mobile apps to connect, learn, play, and work. This lesson will introduce you to the different types of mobile apps, the technologies used to build them, and the fundamental concepts you need to know to start thinking like an app developer.

The Three Basic Types of Mobile Apps

When a business or individual decides to build an app, they must first choose which type of app to create. This decision depends on their budget, target audience, and the features the app requires.

1. Native Apps

A native app is built specifically for a single mobile operating system (OS)—either iOS (for iPhones) or Android. Because they are "native" to the platform, they offer the best performance and have full access to all the device's features, like the camera, GPS, and contact list.

  • Technology Used: Java or Kotlin for native Android apps; Swift or Objective-C for native iOS apps.
  • Pros: Faster performance and a user interface (UI) that feels familiar to the user.
  • Cons: Expensive to build and maintain separate apps for both Android and iOS; updates must be downloaded from the app store; and they take up space on the user's device.

2. Web Apps

A web app is not a real app but a mobile-friendly website that behaves like one. It runs in a web browser like Safari or Chrome and doesn't need to be downloaded or installed from an app store. They are essentially "responsive" websites that adapt their layout to the user's device.

  • Technology Used: HTML5, CSS, JavaScript, Ruby, and other standard web technologies.
  • Pros: Easier to maintain since there is only one codebase, and they work on all devices with a web browser.
  • Cons: Dependent on the browser, require an internet connection to work, and may not be able to access all of the device's hardware features.

3. Hybrid Apps

A hybrid app is a blend of both native and web apps. It is built using web technologies (HTML, CSS, JavaScript) but is then wrapped inside a native "shell" that allows it to be downloaded from an app store and access some device features.

  • Technology Used: A mix of web technologies and native APIs, using frameworks like Ionic.
  • Pros: Often quicker and cheaper to build than native apps and have less code to maintain.
  • Cons: May lack the power and performance of a true native app and may be slower since they have to download web elements.

Cross-Platform Development: The Best of Both Worlds

Building a separate native app for both Android and iOS can be time-consuming and expensive. To solve this problem, cross-platform development frameworks were created. These frameworks allow developers to write code once using a single programming language (often JavaScript) and then compile it into two separate native apps that run on both iOS and Android.

One of the most popular cross-platform frameworks is React Native. It uses JavaScript to build native apps, giving you the performance benefits of a native app with the ease of developing with one codebase. For beginners, the Expo framework is often used with React Native, as it simplifies the development process even further.

Main Things to Consider When Designing an App

A successful app isn't just about code; it's about thoughtful design. Before writing a single line of code, developers think about these key principles:

  • User Interface (UI) vs. User Experience (UX)
    • UI (User Interface): What the user sees — the visual design of the app, including buttons, colors, fonts, and screen layout. The goal of UI is to be visually appealing and clear.
    • UX (User Experience): What the user feels — the overall experience of using the app. Is it easy to use? Is it frustrating? Is it logical? The goal of UX is to make the app efficient, enjoyable, and intuitive.
  • Target Audience: Who are you building this app for? Design differs if it’s for a young child, a business professional, or an elderly person. Understanding your audience helps in decisions about colors, language, and feature complexity.
  • User Flow: The path a user takes through your app to complete a task. For example, to post a photo on Instagram, the flow might be: Open App → Tap '+' button → Select photo → Apply filter → Write caption → Tap 'Share'. Designing clear and simple flows is key to good UX.

Setting Up Your App Development Environment

We will be using React Native with the Expo framework, which lets us build native apps for both iOS and Android using JavaScript.

  1. Install Node.js: This is a JavaScript runtime that Expo uses to build your app.
    Action: Download and install the "LTS" (Long-Term Support) version from the official Node.js website.
  2. Install a Code Editor: We recommend Visual Studio Code (VS Code).
    Action: Download and install it for free from https://code.visualstudio.com/.
  3. Install the Expo Command Line Tool (CLI):
    Action: Open your computer’s Terminal (macOS) or Command Prompt/PowerShell (Windows) and run:
    npm install -g expo-cli
  4. Install the Expo Go App on Your Phone: This free app will run your project live so you can see your changes instantly.
    Action: On your smartphone, go to the App Store (iPhone) or Google Play Store (Android) and install the "Expo Go" app.

What are Libraries?

When developing an app, you don't start from scratch. Developers use libraries, which are collections of pre-written code that can be used to perform common tasks. For example, instead of writing complex code to display a map, a developer can use a map library. This saves time and ensures that standard elements like buttons and video players are reliable.

App Development Exercises

Beginner Coding Exercise: The Simple Counter App

Goal: Build a very simple but fully interactive counter application. This project will teach you the fundamentals of managing a value that changes based on user input.

What the App Does & Looks Like: The app displays a number prominently in the center of the screen with two buttons below it: "Increase" and "Decrease". Pressing "Increase" increments the number by one, and pressing "Decrease" decrements it by one. The interface should be clean and simple.

Challenge Coding Exercise: Build a To-Do List App

Your Goal: Build a functional to-do list application that manages a list of tasks changing over time based on user input.

What the App Does & Looks Like:

  • Input Area: At the top, a text input field for typing a new task and an "Add Task" button.
  • Task List: Below the input area, a list displaying all added tasks. When a task is added, it should appear instantly in the list.
